# Coobie Causaloid Roadmap

Purpose: move Coobie from heuristic after-action diagnosis toward reusable causal reasoning that spans application behavior, runtime environment, and regulatory context.

Priority items:
- Model application-level and environment-level causes together instead of treating them as separate stories.
- Add industrial signals for timing, burst data loss, PLC acknowledgement drift, historian lag, SCADA alarm semantics, simulator fidelity, and degraded-mode transitions.
- Treat regulatory constraints such as GMP, GxP, audit trails, and validation evidence as first-class causal factors rather than late reporting concerns.
- Promote active causaloids into preflight guardrails so Coobie affects planning before Scout, Mason, Ash, Bramble, and Sable act.
- Evolve from simple scalar thresholds toward richer contextual hypergraphs and policy-aware propagation when the domain requires it.

Expected outcome:
- Coobie can explain not only what failed, but which environmental conditions and policy constraints made the failure likely and which intervention should be tried next.
