# Coobie Causaloid Roadmap

Purpose: move Coobie from heuristic after-action diagnosis toward reusable causal reasoning that spans application behavior, runtime environment, regulatory context, and worker-plan quality.

Priority items:
- Align Coobie's causal model with DeepCausality's current primitive: effect propagation (`E2 = f(E1)`), not only labeled cause/effect edges.
- Model application-level and environment-level causes together instead of treating them as separate stories.
- Promote stable Calvin `causal-pattern` records into executable causaloids with input projection, context requirements, activation predicate, expected effect, and explanation path.
- Represent run/spec/agent/chamber/time/evidence/provider/OpenZiti posture as explicit contextoids so causal functions can test assumptions instead of reading prose summaries.
- Use Pearl labels as evidence levels: PackChat `causation_id` starts associational, explicit run changes can promote to interventional, and named alternate effect paths can promote to counterfactual.
- Map Keeper and the Calvin Meta-Governor to DeepCausality's Effect Ethos role: causal recommendations must pass deontic and identity checks before becoming Praxis actions.
- Add industrial signals for timing, burst data loss, PLC acknowledgement drift, historian lag, SCADA alarm semantics, simulator fidelity, and degraded-mode transitions.
- Treat regulatory constraints such as GMP, GxP, audit trails, and validation evidence as first-class causal factors rather than late reporting concerns.
- Promote active causaloids into preflight guardrails so Coobie affects planning before Scout, Mason, Ash, Bramble, and Sable act.
- Critique worker plans before execution: if a planner ignores known causal hazards, stale memories, or hidden-scenario failure modes, Coobie should block or revise the plan.
- Track worker-harness failures as causal objects too: insufficient exploration, unverified execution, poor tool isolation, and weak evidence should become reusable cause categories.
- Evolve from simple scalar thresholds toward richer contextual hypergraphs and policy-aware propagation when the domain requires it.
- Before DeepCausality Phase 2, review whether to migrate from Harkonnen's current `deep_causality = "0.3"` dependency to the current modular DeepCausality crate stack.

Expected outcome:
- Coobie can explain not only what failed, but which environmental conditions, worker-plan decisions, and policy constraints made the failure likely and which intervention should be tried next.
